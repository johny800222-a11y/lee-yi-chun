## 一、系統概觀

這套系統是「零 API 虛擬單系統」，用**真實市場行情數據**驅動策略邏輯，
但完全不連接交易所下單 API，所有倉位與資金都存在本地 JSON 檔案中模擬。

目的：在觀察期（30天）用真實市場驗證策略，確認有效後再接真實交易。

## 二、目前運行中的三套策略

### 1. EMA99 策略（趨勢追蹤）
- **邏輯核心**：EMA99 作為趨勢過濾，只做順勢方向
- **掃描頻率**：每 15 分鐘（15m K 線收盤觸發）
- **掃描範圍**：市值前 100 的 USDT 永續合約（排除穩定幣）
- **進場條件**：
  - 做多：收盤價 > EMA99 且 EMA99 斜率向上
  - 做空：收盤價 < EMA99 且 EMA99 斜率向下
  - 量比 > 0.8（量縮不進場）
  - RSI 未極端（做多 < 75，做空 > 25）
- **出場**：TP1=2R(33%), TP2=3.5R(50%), TP3=5.5R(全出) / SL=結構位
- **狀態檔**：`ema99_bot_state.json`

### 2. NFES 策略（多因子過濾趨勢）
- **邏輯核心**：N(趨勢) + F(假突破) + E(情緒/資金費率) + S(強度)
- **掃描頻率**：每 1 分鐘（主循環）
- **掃描範圍**：市值前 100 的 USDT 永續合約
- **進場條件**（四項同時成立）：
  - 15m + 1H 雙時框方向一致
  - RSI 未極端（40-60 區間外才考慮，>70 空 / <30 多）
  - 資金費率未極端（極端時反向）
  - 多空比配合方向
- **出場**：與 EMA99 相同三段出場
- **狀態檔**：`nfes_bot_state.json`

### 3. Phantom 策略（逆勢反轉，觀察期中）
- **邏輯核心**：捕捉「止損獵殺後的反轉」
- **觀察期**：2026/05/19 ~ 2026/06/19（30天）
- **三合一條件**：
  1. 資金費率極端（多頭費率 > +0.1% 做空反轉 / 空頭費率 < -0.1% 做多反轉）
  2. False Breakout（突破前高/低後快速回收）
  3. 多空比極端（散戶情緒偏向一方到頂）
- **精髓**：主力獵殺止損完成後，方向反轉的那個瞬間
- **狀態**：目前仍在觀察研究中，每週一 12:30 發週報

## 四、零 API 虛擬單原理

### 使用的 API（完全公開，無需帳號）
```
Binance 公開行情 API：
GET /fapi/v1/klines        → K線數據
GET /fapi/v1/markPrice     → 當前標記價格
GET /fapi/v1/ticker/24hr   → 24h量、漲跌幅
GET /fapi/v1/fundingRate   → 資金費率（Phantom 用）
GET /fapi/v2/ticker/bookTicker → 最新買賣價
```

### 不使用的 API
```
POST /fapi/v1/order  → 下單（需帳號 + 簽名 + 資金）
GET /fapi/v2/account → 帳戶資訊（需 API Key）
```

### 虛擬倉位生命週期
```
進場：
  positions[symbol] = {
    side: "long"/"short",
    entry_px: 當前標記價,
    size: notional / entry_px,
    sl: 結構位止損,
    tp1/tp2/tp3: 目標價,
    margin: notional * (1/leverage)
  }
  capital -= margin  ← 凍結保證金

持倉中（每分鐘）：
  cur = 最新標記價
  if tp1_hit: 出場 33%，SL 移保本
  if tp2_hit: 出場再 50%，SL 移 TP1
  if tp3_hit / sl_hit: 全出

出場：
  pnl = (exit_px - entry_px) / entry_px * notional  (多)
  pnl = (entry_px - exit_px) / entry_px * notional  (空)
  capital += margin + pnl  ← 歸還保證金 + 損益
  records trade in trades[]
```

## 五、訊號邏輯（為什麼這週沒有進場）

### 市場橫盤/上漲時：
- EMA99 斜率為正 → 封鎖做空
- NFES RSI 約 45（中性）→ 多空方向不明
- 缺乏多時框趨勢對齊 → 所有訊號被過濾

**哲學**：寧可錯過，不做錯。量縮 / 方向不明 = 不進場。

### 真正的進場時機：
- 市場有明確方向（EMA99 明顯斜率）
- 多時框對齊（15m + 1H 同向）
- 有結構位可放止損
- 量比 > 1.0（有人在動）

## 六、Phantom 策略的核心假設

```
假設：
  主力的獵殺行為是可預測的
  └→ 當散戶止損集中在某個位置
  └→ 主力會先推價格到那裡（觸發止損）
  └→ 然後快速反轉（因為止損掃完了）

Phantom 就是在「止損被掃完」的那一刻進場

三個確認信號缺一不可：
  1. 資金費率極端 → 確認散戶情緒偏向一側（即將被清洗）
  2. False Breakout → 確認獵殺行為發生（突破後快速縮回）
  3. 多空比極端 → 確認散戶倉位已過度集中（主力有理由獵殺）
```

## 七、關鍵技術細節（給大腦參考）

### 精度問題（已修復）
某些低價幣（如 DOGS，價格 5.498e-05）在計算 TP 時：
```
round(0.000055, 2) = 0.0  ← 精度丟失
tp3 = 0.0
tp3_hit = (cur >= 0.0) = 永遠 True
→ 立即以 0 出場 → 損失 100% 本金
```
修復：改用 `_smart_round()` 保留有效位數而非固定 2 位小數。

### 績效追蹤
- `total_realized_pnl`：累計實現盈虧（不受交易筆數限制影響）
- `total_trades`：累計交易次數
- `total_wins`：累計獲利次數
- `trades[-500:]`：保留最近 500 筆（原本 50 筆會丟失早期數據）
